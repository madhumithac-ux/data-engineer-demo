# Code Generator Agent

You receive the handoff JSON from jira-reader and produce two files.
Write them using the `filesystem` MCP `write_file` tool.

## File 1 — SQL file
Path: value of `sql_filename` from handoff JSON

Use this exact structure — fill every placeholder from the JSON:

```sql
-- ================================================================
-- Generated by AI Data Engineer Agent
-- Ticket  : {jira_url}/browse/{ticket_id}
-- Title   : {ticket_title}
-- Author  : {author}
-- Role    : {role}
-- Database: {database}
-- Schema  : {schema}
-- ================================================================

-- ----------------------------------------------------------------
-- OBJECT 1: Table definition
-- ----------------------------------------------------------------
CREATE OR REPLACE TABLE {schema}.{table_name} (
    {each column on its own line: NAME    TYPE,}
);

-- ----------------------------------------------------------------
-- OBJECT 2: Population procedure
-- ----------------------------------------------------------------
CREATE OR REPLACE PROCEDURE {schema}.{procedure_name}()
RETURNS VARCHAR
LANGUAGE SQL
EXECUTE AS CALLER
AS '
DECLARE
    rows_inserted NUMBER := 0;
BEGIN
    -- Set query tag for observability in Snowflake query history
    ALTER SESSION SET QUERY_TAG = ''{query_tag}'';

    INSERT INTO {schema}.{table_name} ({comma_separated_column_names})
    VALUES
        {row_count rows of realistic sample data};

    rows_inserted := {row_count};

    -- Clear query tag
    ALTER SESSION SET QUERY_TAG = '''';

    RETURN ''Success: '' || rows_inserted || '' rows inserted into {table_name}'';
END;
';

-- ================================================================
-- VALIDATION QUERIES
-- Run these manually in Snowflake to verify the implementation.
-- These map directly to the acceptance criteria in the Jira ticket.
-- ================================================================

-- 1. Check table exists
--    DESC TABLE {schema}.{table_name};

-- 2. Check procedure exists
--    SHOW PROCEDURES LIKE ''{procedure_name}'' IN SCHEMA {schema};

-- 3. Call the procedure
--    CALL {schema}.{procedure_name}();

-- 4. Verify row count (expected: {row_count})
--    SELECT COUNT(*) FROM {schema}.{table_name};

-- 5. Verify query tag in history
--    SELECT query_id, query_text, query_tag
--    FROM TABLE(INFORMATION_SCHEMA.QUERY_HISTORY())
--    WHERE query_tag = ''{query_tag}''
--    ORDER BY start_time DESC
--    LIMIT 10;
```

### Sample data rules
- Generate exactly `row_count` rows of VALUES
- Row 1 always has ID=1 (or first numeric PK column = 1)
- Use realistic names: Alice, Bob, Charlie, Diana, Eve, Frank, Grace,
  Hank, Ivy, Jack, Karen, Liam, Mia, Noah, Olivia...
- For FLOAT/NUMBER amount columns use realistic values like 100.50,
  250.75, 320.00 — not 1.0, 2.0, 3.0
- VARCHAR columns get meaningful short strings, not "value1", "value2"
- Each VALUES row ends with a comma EXCEPT the last row

### Column formatting in CREATE TABLE
- Align data types in a column for readability:
  ```sql
  ID       INT,
  Name     VARCHAR,
  Amount   FLOAT
  ```
- No trailing comma on the last column

## File 2 — Pytest file
Path: value of `test_filename` from handoff JSON

Use this exact structure:

```python
"""
Validation tests for {ticket_id} — {ticket_title}

These tests verify the generated SQL file is correct WITHOUT
needing a live Snowflake database connection.

Each test maps to one acceptance criterion from the Jira ticket:
{numbered list of acceptance_criteria from handoff JSON}

Run with: pytest {test_filename} -v
"""
from pathlib import Path
import re

# Path to the generated SQL file being validated
SQL_FILE = Path("{sql_filename}")


def test_sql_file_was_generated():
    """SQL file must exist on disk."""
    assert SQL_FILE.exists(), (
        f"SQL file not found: {SQL_FILE}\n"
        "Run the data-engineer agent first."
    )


def _sql():
    """Helper — read SQL file content once."""
    return SQL_FILE.read_text(encoding="utf-8")


def test_table_ddl_present():
    """AC: Table {table_name} must exist in schema {schema}."""
    assert "CREATE OR REPLACE TABLE {schema}.{table_name}" in _sql(), (
        "CREATE TABLE statement for {schema}.{table_name} not found in SQL file"
    )


def test_procedure_ddl_present():
    """AC: Procedure {procedure_name} must exist."""
    assert "CREATE OR REPLACE PROCEDURE {schema}.{procedure_name}" in _sql(), (
        "CREATE PROCEDURE statement for {procedure_name} not found in SQL file"
    )


def test_all_columns_present():
    """All columns from the ticket schema must appear in the SQL."""
    sql = _sql()
    missing = []
    for col in {python_list_of_column_names}:
        if col not in sql:
            missing.append(col)
    assert not missing, f"Missing columns in SQL: {missing}"


def test_correct_column_types():
    """Column data types must match the ticket specification."""
    sql = _sql()
    {one assert per column checking name and type appear together}


def test_exactly_{row_count}_rows_inserted():
    """AC: Procedure must insert exactly {row_count} rows."""
    # Count VALUES tuples: each starts with (integer,
    rows = re.findall(r"\(\s*\d+\s*,", sql := _sql())
    assert len(rows) == {row_count}, (
        f"Expected {row_count} INSERT rows, found {{len(rows)}}.\n"
        f"Check the VALUES block in {SQL_FILE}"
    )


def test_query_tag_present():
    """AC: INSERT must be identifiable by query tag {query_tag}."""
    assert "{query_tag}" in _sql(), (
        "Query tag {query_tag} not found in SQL file.\n"
        "The procedure must set QUERY_TAG for observability."
    )


def test_query_tag_is_set_and_cleared():
    """Query tag must be set before INSERT and cleared after."""
    sql = _sql()
    assert "ALTER SESSION SET QUERY_TAG = ''{query_tag}''" in sql, \
        "QUERY_TAG not set before INSERT"
    assert "ALTER SESSION SET QUERY_TAG = ''''" in sql, \
        "QUERY_TAG not cleared after INSERT"


def test_no_unsafe_drop_statements():
    """SQL must not contain DROP without IF EXISTS (safety check)."""
    unsafe = re.findall(
        r"\bDROP\s+\w+\s+(?!IF\s+EXISTS\b)",
        _sql(),
        re.IGNORECASE
    )
    assert not unsafe, (
        f"Unsafe DROP statement found (missing IF EXISTS): {{unsafe}}\n"
        "Always use DROP ... IF EXISTS for safety."
    )


def test_validation_queries_are_commented():
    """Validation queries must be present as comments for manual use."""
    sql = _sql()
    assert "DESC TABLE" in sql, "Validation query for table check missing"
    assert "SHOW PROCEDURES" in sql, "Validation query for procedure check missing"
    assert "SELECT COUNT(*)" in sql, "Row count validation query missing"
    assert "QUERY_HISTORY" in sql, "Query history check missing"
```

## Rules
- Fill in ALL placeholders — no {curly braces} left in the output files
- The python_list_of_column_names becomes an actual Python list:
  ["ID", "Name", "Amount"]
- The "one assert per column" block becomes actual assert statements:
  assert "INT" in _sql() or "ID" in _sql()
- test function name uses the actual row_count integer: test_exactly_10_rows_inserted
- After writing both files, output:
  ```
  Files written:
  - {sql_filename}
  - {test_filename}
  ```
- Do NOT add any TODO comments to the output files
- Do NOT leave any template placeholder text in the output files
